talk-html · 一个 Claude Code 技能 2026-05-14

用 HTML 说话，
不要用聊天滚动条。

talk-html 是一个 Claude Code 技能。它把 agent 的长回答变成一页打磨过的 HTML——本地先预览，发布后留存，几周后还能凭一个关键词找回来。装一次，之后在对话里说一句"做成一页"就够了。

一行装好
curl -fsSL https://raw.githubusercontent.com/LiuShiyuMath/talk-html-skill/main/install.sh | bash

装进 ~/.claude/skills/talk-html/。装好后在 Claude Code 里用 /talk-html 唤起，或直接说"用 html 解释一下"。

Why

它解决一个具体的麻烦。

并行开着好几个 Claude 会话时，agent 产出的 HTML 一份份散落在 ~/.claude/jobs/、worktree、/tmp 里。每份刚做出来都有用，隔几天就忘了从哪儿来。talk-html 把三件事收进一个技能：

  • 一 沟通。把长回答渲染成有版式、能读的一页 HTML，而不是让人在聊天框里上下翻。
  • 二 留存。本地预览满意后，一键发布到 gist（或像这一页一样发到 GitHub Pages），拿到可分享的链接。
  • 三 回看。每份产出都进一个本地索引，凭关键词就能翻出来——这是大多数技能会忘掉的那一件。
How it works

整条路径一句话讲完。

本地生成 → 浏览器预览 → 跟你确认 → 发布留存 → 写进索引。几周后用一个词找回来。

预览那一步不能省——你几乎不想把没自己看过的页面发出去。
Triggers

装好之后，怎么叫它。

用斜杠命令，或者在对话里直接说下面任意一句。agent 会自己选版式、写 HTML、在浏览器里打开，等你点头：

  • 用 html 解释
  • 用 html 说
  • 做个网页
  • 做成一页
  • 推到 gist
  • html 回看
  • html 备忘
  • 解释给＜人＞看
  • /talk-html
  • talk in html
  • make a page out of this
  • give me an html version
What you get

四个文件，各做一件事。

~/.claude/skills/talk-html/ ├── SKILL.md # agent 读的工作流与质量标准 ├── publish.sh # 发布到 gist + 写回看索引 ├── recall.sh # 列出 / 按关键词打开旧产出 └── templates/ └── skeleton.html # 必备结构槽位（不规定视觉）

产出统一落在 ~/.claude/talk-html/，旁边的 index.jsonl 每发布一份记一行。想找回某一页，跑 recall.sh ＜关键词＞ 就在浏览器里打开了。
